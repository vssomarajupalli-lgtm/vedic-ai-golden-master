# ADR-010: Formula Scaling via Many-to-One Convergence Rule

## Context
During Phase 14, as the Question Catalog expanded significantly, there was a risk of creating duplicate mathematical logic for semantically similar questions. A rule was required to govern how multiple frontend Question IDs are resolved into backend formula logic.

## Repository Evidence
"- Confirmed via the mapping constraint ensuring ~500 Question IDs converge down to ~100 Formula Variants, which in turn inherit from ~35 Base Formula Families."

## Decision
New UI Question IDs must aggressively map to existing formula variants instead of creating 1-to-1 duplicated math logic whenever the astrological math is identical.

## Architectural Rationale
This prevents formula explosion and repository bloat. Without it, developers may incorrectly create duplicate math definitions for semantically similar questions, causing massive maintenance burdens.

## Consequences
Formula variants are reduced, promoting reuse. The Question Registry connects multiple natural language queries to identical Formula Variants.
