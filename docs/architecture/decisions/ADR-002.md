## Decision
The Question Registry (`question_registry.json`) shall be maintained as a strict JSON array.

## Architectural Rationale
A JSON array facilitates rapid O(1) hash mapping during API routing. It provides a lightweight, highly performant mapping layer that is completely separated from the complex YAML formula logic used for astrological rules.

## Consequences
The UI and API routing layer depend on this specific JSON structure. Schema adherence (e.g., `question_id`, `formula_key`) is rigorously enforced at runtime.
