# ADR-015: Graceful Degradation via MIXED State Capping

## ADR ID

## Title
Graceful Degradation via MIXED State Capping

## Status
Accepted

## Recovery Classification
Recovered during GM-005 Repository Refinement

## Context
In scenarios where specific downstream engines (e.g., TransitEngine) experience failures or timeouts, the system required a resilient methodology to prevent complete crashes while avoiding false-positive outputs.

## Repository Evidence
"- Established the protocol for capping results at `MIXED` if a required engine (like `TransitEngine`) fails, mitigating systemic outages."

## Decision
If an external engine drops out or fails during evaluation, the system must gracefully degrade and cap the overall result at a `MIXED` state, rather than generating a false positive or triggering a complete crash.

## Architectural Rationale
Documents a critical resilience mechanism for production stability. It ensures that the system handles missing data defensively and safely.

## Consequences
System failures in specific evaluation layers result in a default to a safe `MIXED` outcome. This guarantees a safe degradation of service without misleading the user.
