## Context
During Phase 9, reliance on Generative NLP to determine user intent proved fragile and prone to routing errors, requiring a pivot to a mathematically locked structure.

## Repository Evidence
"The hierarchical data dictionary strictly binds frontend user queries to backend mathematical engine logic, replacing fragile NLP intent matching with a deterministic mapping matrix."

## Decision
The application shall explicitly use hard-mapped IDs (a deterministic mapping matrix) rather than Generative NLP to route a user's question to the corresponding mathematical formula.

## Architectural Rationale
This massive architectural shift prioritizes safety and determinism over generative query flexibility. It ensures that asking the same exact query will always trigger the exact same mathematical formula pipeline without fail.

## Consequences
NLP intent resolution is structurally removed from the critical routing path. The UI must pass strict, predefined Question IDs to the backend APIs.
