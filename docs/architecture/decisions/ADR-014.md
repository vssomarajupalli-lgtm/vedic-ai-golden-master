## Context
To prevent astrological hallucinations and ensure that the generative UI layers do not invent answers outside the scope of the pre-calculated formulas, specific firewall rules needed to be established for the Answer Composer API.

## Repository Evidence
"- Defined the firewall rules for the Answer Composer to prevent LLM hallucinations (Context Injection bounding, Evaluation Overriding prohibition, Tone enforcement)."

## Decision
The Answer Composer API must enforce specific input-bounding firewall boundaries: Context Injection bounding, Evaluation Overriding prohibition, and Tone enforcement.

## Architectural Rationale
This defines the critical safety perimeter guarding LLM usage in production astrology. It ensures the LLM generates text strictly based on provided facts without extrapolating beyond the injected context.

## Consequences
The Answer Composer is constrained by rigid input parameters. The prompt is constructed exclusively from the payload defined by the Formula.
