# ADR-001: YAML as the Canonical Formula Storage Format

## ADR ID

## Title
YAML as the Canonical Formula Storage Format

## Status

## Context
During the Phase 10 integration, the system needed a scalable, decoupled format for storing astrological formula configurations that prevented logic hardcoding. A decision was made to select a canonical storage format for the Formula Repository.

## Repository Evidence
"**YAML** is the strictly recommended format for the Formula Repository... A Python registry (`formulas.py`) invites the temptation to write executable logic inside the definitions, violating architectural decoupling."

## Decision
Formula Repository configurations shall be stored exclusively in YAML format.

## Architectural Rationale
YAML allows for inline comments explaining astrological reasoning (e.g., why a specific Bhava or Yoga is included in a formula). It forces formulas to remain purely declarative, completely decoupling astrological definitions from Python execution logic.

## Consequences
Developers cannot inject Python code into formula definitions. Future loader classes shall parse YAML into dictionaries instead of executing Python modules.

## Compliance Notes
Any attempt to revert formulas back to Python dictionaries or executable scripts shall be rejected. The decoupling of math execution from formula definition relies on this declarative format.
